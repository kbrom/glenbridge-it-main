Technical Implementation

The website shutdown was implemented using a multi-layered approach:

  1. Removed all original website content from S3 bucket glenbridge-it.com
  2. Created and uploaded a permanent closure notice as the main page
  3. Added closure pages to all subdirectories to catch direct URL access
  4. Created error pages for all common HTTP error codes
  5. Configured S3 website redirection rules to redirect all errors to the main closure page
  6. Added robots.txt to prevent search engine indexing
  7. Set cache-control headers to prevent browser caching
  8. Executed multiple CloudFront invalidations to purge cached content

While we couldn't disable the CloudFront distributions due to permission limitations, the implemented measures ensure the website displays only the closure notice and can be considered effectively shut down.
